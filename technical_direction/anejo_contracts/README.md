# Anejo Contracts

Esta carpeta define contratos minimos por anejo para orientar produccion premium, revision de coherencia y validaciones.

No sustituye al anejo ni a su fuente tecnica.

Su funcion es fijar:

1. datos minimos que deben existir;
2. salidas premium recomendadas;
3. validaciones de cierre o de coherencia;
4. limites de lo que no debe inventarse.

Reglas:

- si no existe fuente tecnica suficiente, el contrato debe dejar pendientes visibles;
- los contratos deben seguir `FUENTES_MAESTRAS.md`;
- si un contrato madura y demuestra reutilizacion, debe promocionarse a `urbanizacion-toolkit`.
